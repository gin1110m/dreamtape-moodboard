# dreamtape — Art Direction Guide / Moodboard

dreamtape（Satoru × Ginpei の音楽プロジェクト）のアートディレクション・ガイド兼ムードボード。
**白地 16:9 のHTMLスライドデッキ1枚**で、GitHub Pages に公開している。

- **公開URL**: https://gin1110m.github.io/dreamtape-moodboard/
  共有時はキャッシュ回避で `?v=<short-sha>` を付ける（例 `?v=b8c97ee`）。`?lang=ja` で日本語、`?lang=en` で英語に直リンク。
- 二人の監督（Satoru × Ginpei）が共同編集する。これは**社内/二人の間の資料**。

## ファイル構成
- `index.html` — **デッキ全体。CSS・JS・i18n辞書まで全部この1ファイルに入っている**（自己完結。外部依存はGoogle Fonts CDNのみ）。編集は基本ここ。
- `assets/` — 画像（`*.jpg`）。デッキ内は相対パス `assets/...` で参照。
- `scripts/gen_i18n_zwsp.py` — 日本語テキストの生成スクリプト（後述）。

## プレビュー / 公開
- **プレビュー**: `index.html` をブラウザで直接開くだけ（ローカルサーバ不要）。
- **公開**: `git push`（main）→ GitHub Pages が約1分で自動デプロイ。新URLは `?v=<コミットshaの先頭7桁>` を付けて共有する（素のURLはエッジキャッシュで古い版が出ることがある）。
- 画像を足すときは `assets/` にファイルを置き、`index.html` から `assets/ファイル名` で参照する。

## ブランドDNA（守る）
- 「音を聴いた瞬間に、景色が見える」／ tape your dream ／ 作品=映画・音楽=サウンドトラック。
- **Magician 80% + Everyman 20%**：世界をつくるが種明かしはしない。でも目線は聴き手と同じ。
- Vintage・Analog・Organic・Dry／Cinematic Vintage／温かく褪せたフィルム感。
- **ラグジュアリーの誇示・hype はNG**（「高そうに見せる」「煽る」をしない）。上質さは"抑制"で出す。
- **常に小文字 `dreamtape`**。言葉は最小限。
- 人物は**顔を出さない**（逆光・影・半分隠れる＝Magician）。

## デザインシステム（Cult Creatives の Art Direction Playbook 方式に準拠）
- 色: 紙 `#f7f4ee` ／ インク `#191712` ／ アクセント=テラコッタ `#c4683d`（**1色のみ**）／ スチール `#5b6b78`。CSS変数は `:root` 参照。
- 書体: **Bebas Neue**=全見出し＋巨大トラッキングのキーワード（DUSK / DREAM / QUIET 等）。**Helvetica Neue（無ければArimo）**=ワードマーク `dreamtape.`（小文字）＋本文＋ラベル＋メタ。
- 写真ページの型（Cult式・実フレーム観察済）:
  - **行型**=画像1行＋下に巨大キーワード＋**左下に円形バッジ**（このデッキでは `dt.`）＋右に小注記。→ `REF II`（`.board-cult`）。
  - **コラージュ型**=混在サイズの密集ピンボード＋右に本文コラム（キーワード無し）。→ `REF I`（`.wall` / `.wall-imgs` を6×4グリッドに `nth-child` でspan配置）。
- REF（リファレンス）ページは **"FOUND PHOTOGRAPHY（拾い物の実写・自分たちの画像ではない）"** と明示する（`[ REFERENCE ONLY ]` / `NOT OUR IMAGES`）。最終成果物ではなく"めざすトーン"の参照。

## 日本語 / i18n の編集（重要・手順を守る）
- 英語テキストは `index.html` に直書き。日本語は同ファイル末尾の `const I18N = {...}` 辞書に入っていて、**`scripts/gen_i18n_zwsp.py` から自動生成**している（BudouXで文節ごとにZWSPを挿入し、変な位置で改行されないようにしている）。
- **日本語を直すとき**: `index.html` の `I18N` ブロックを手で触らない。代わりに `scripts/gen_i18n_zwsp.py` の `DICT` を直して、次を実行:
  ```bash
  pip install budoux          # 初回のみ
  python3 scripts/gen_i18n_zwsp.py   # I18Nブロックを再生成（ZWSP込み）
  ```
- 翻訳要素は HTML側に `data-i18n="キー"` が付いている。新しく訳したい要素にキーを付け、`DICT` に対訳を足して再生成する。
- **ブランド資産はラテンのまま**（ワードマーク／巨大Bebasキーワード／画像コード／スペック）。`data-i18n` を付けないこと。和訳は**平易に**（硬い漢語を避ける）。

## "壊すな" ルール（過去に踏んだ地雷）
- `.wall`（コラージュ）の単一行は **`grid-template-rows:minmax(0,1fr)` ＋ 子に `min-height:0`** が必須。無いと行が高さを取れず画像が縦に溢れる。
- **地名を地の文に書かない**（LA / Los Angeles / 座標）。雰囲気=cinematic vintageは出すが、地名は出さない。
- **ブロックを「左の色付き縦ボーダー」で囲まない**（AIっぽさの典型）。色分けはピル/テキスト/数値に逃がす。
- 並列ブロックは**見出し帯の高さ・縁取り・影を完全に揃える**。
- 可読性優先（薄い細い小さい文字を避ける／濃いインク＋中字以上）。
- **各ページに固有の意味を持たせる**。似た見た目のページを並べない（過去にREFを同型4連発にして「違いが分からない」と指摘され、コラージュ1枚＋行1枚に集約した経緯あり）。
- 公開リポジトリなので **APIキー・認証情報・社外秘の戦略は置かない**。
